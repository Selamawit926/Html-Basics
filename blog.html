Question No.4

Question: Does it matter to use relative size(%,em,rem) one styling the size of an element so that the element is responsive? Why? [write your argument]

Yes, it definitely matters to use percentage, em, and rem units to make elements more responsive. Since em is a flexible unit that is converted by the browser into pixel values and is relative to the font size of the parent element, for a more responsive layout all the user need to do is change the default font size. For example, if the default is 16px and that of a paragraph is 2em then it’ll be translated into 32px (twice of the parent). This particular relative unit also has an inheritance property, which means if there was another div inside the paragraph with a font size of 1.5m then it’ll translate into 48px (not 24px). It’ll inherit the font size of its nearest parent as the default. This can be solved by using rem relative unit. Rem is relative to the font size of the root element (html element). If the font-size of the html element is 16px, then 1rem = 16px. It’ll always be relative to the root element despite how nested the elements are. Percentage is similar to em in that they are relative to the parent’s size and are widely used in responsive layouts.

When we look at the uses of the relative units, they come in handy when dealing with the zooming features of browsers. The default font size of a browser is usually 16px and when using pixels and ems for paragraphs or headers font size, there is a significant difference between them while zooming in and out. For instance, let’s assume we have two paragraphs, one with font size with 24px and the other with 2rem while the default is 16px. If we were to zoom out of the page, the first paragraph will remain the same since it has static value in pixels and therefore will seem to decrease in size. The second paragraph has a responsive unit in rems so the font size will scale up nicely since it’ll be twice the default size.

Generally, the use of relative units like percentage, em and rem on the style of an element is encouraged because they make it responsive and flexible for the user.

Question No.5

Question: We said 16px is default browser width so what does it mean when we say 200px? How does the browser interpret that?

When we say 200px we’re actually telling the browser to change the default font size from the 16px and it’s going to interpret that as overriding its default. So the rest of the elements are going to inherit from the given size instead of the original default font size of the browser.

Question No.6

Question: Explain in detail(in technical terms) the specific changes in css3.

The CSS3 version has added new features to help developers style html elements using less code. A few of the new features include:

  • Box shadow: it adds a shadow to an element. Instead of using multiple images around an item, this property lets you add shadow with a short line of code.
  • Opacity: this makes elements see-through or completely transparent. It allows developers to apply opacity to images or other HTML elements. The transparency level depends on the indicated values.
  • Rounded corners: it allows developers to apply the border-radius CSS3 property to HTML elements so that the element can turn out rounded or as desired.
  • Attribute selectors: this allows developers to select HTML elements according to their attributes instead of applying Id’s or classes for styling. It saves them from creating unique IDs only to apply CSS rules.
  • New colors: One of the CSS3 features is the addition of new colors like RGBA, HSL, HSLA and gradient colors.
  • More than web-safe fonts: this allows developers to apply more unique fonts in their HTML documents instead of only using fonts labeled as web-safe.
  • Animations and Transitions: there are two ways to create CSS animations. The first is very easy, it is done through animating the changes of CSS properties with the transition declaration. With transitions, you can create hover or mouse down effects, or you can trigger the animation by changing the style of an element with JavaScript. The second way for defining animations is a bit more complicated - it involves the description of specific moments of the animation with the code. This allows you to have repeating animations that don't depend on user actions or JavaScript to get triggered.
  • The calc() function: it allows developers to do simple arithmetic calculations in CSS. They can use it anywhere a length or a size is required.
  • 3D transforms: this offers some powerful features to designers and developers. It allows elements to be shifted, rotated, slanted, squashed and stretched.
  • Media Queries: they're like an extended version of the media types, which were introduced in CSS2. They can be used to check many things including the device resolution and orientation, color depth, pixel density , wdth and height of the device and more.
